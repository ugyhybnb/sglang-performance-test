# 产物清单

这份清单用来回答一个重要问题：

这个仓库里到底哪些内容是“展示材料”，哪些内容是“复现材料”，哪些内容是“原始证据”。

## 1. 建议优先阅读

如果你是第一次访问这个仓库，推荐顺序如下：

1. [README.md](README.md)
2. [项目A-公开摘要.md](项目A-公开摘要.md)
3. [项目A-v2-最终结果文档.md](项目A-v2-最终结果文档.md)
4. [项目A-v2-简历描述.md](项目A-v2-简历描述.md)

## 2. 展示材料

这些文件主要服务于 GitHub 首页、GitHub Pages 和个人网站展示：

- `README.md`
- `docs/index.md`
- `docs/assets/*.png`
- `figures/final/*.png`
- `项目A-公开摘要.md`
- `项目A-v2-简历描述.md`

特点：

- 易读
- 结果导向
- 适合快速浏览

## 3. 复现材料

这些文件主要服务于“别人如何按步骤复现你的方法”：

- `项目A-复现说明.md`
- `项目A-v2-服务基线.md`
- `scripts/run_bench_case.sh`
- `scripts/run_mixed_case.sh`
- `scripts/run_mixed_traffic.py`
- `scripts/plot_projectA_v2.py`

特点：

- 包含命令
- 包含输入输出路径
- 包含脚本入口

## 4. 技术报告材料

这些文件主要服务于深入阅读和面试追问：

- `项目A-v2-最终结果文档.md`
- `项目A-v2-request-rate压测.md`
- `项目A-v2-max-concurrency压测.md`
- `项目A-v2-长prompt专项.md`
- `项目A-v2-长输出专项.md`
- `项目A-v2-shared-prefix专项.md`
- `项目A-v2-混合流量专项.md`
- `项目A-v2-长时稳定性专项.md`
- `项目A-v2-可视化说明.md`

特点：

- 信息完整
- 能支撑面试细问
- 能解释为什么得出这些结论

## 5. 原始证据材料

这些内容最能体现项目真实性，但不一定适合全部放在首页：

- `jsonl/`
- `logs/`
- `metrics/`
- `snapshots/`

特点：

- 体积更大
- 可读性较低
- 但对追溯实验过程很重要

## 6. 当前公开策略

当前仓库采取的是“展示优先、证据保留、首页减噪”的公开策略：

- 首页突出结论、图表和项目价值
- 复现说明保留方法
- 脚本保留入口
- 原始证据作为项目背书保留

这比把所有采样文件直接堆在首页更适合作为名片项目。

## 7. 是否还要继续补内容

如果你希望仓库更像典型开源项目，而不仅是个人作品集项目，后续还建议补：

- `LICENSE`
- `CHANGELOG.md`
- 中英文双语 README
- GitHub Release，把大体积原始数据打包成附件

这样可以进一步提升公开仓库的完成度和可信度。
